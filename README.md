features/ — по функционалу (сейчас telegram, позже добавить auth/registration, users, billing и т.д.).
	•	infra/ — всё «железо» (логгер, сторедж, безопасность). Легко менять реализации.
	•	shared/ — типы, ошибки, схемы, которые переиспользуются в нескольких фичах.
	•	routes/ — только объявления маршрутов (тонкий слой).
	•	config/ — .env и конфигурация в одном месте.
	•	app/ — запуск и сборка приложения.
